## Template Notes (DELETE THIS SECTION)

### Version Header Format

```
## [x.y.z] - YYYY-MM-DD
```

### Categories (in order)

1. **Added** - New features
2. **Changed** - Changes in existing functionality
3. **Deprecated** - Soon-to-be removed features
4. **Removed** - Now removed features
5. **Fixed** - Any bug fixes
6. **Security** - Vulnerability fixes

### Tips

- Write entries in imperative mood ("Add feature" not "Added feature")
- Include issue/PR numbers when relevant: `Fix login bug (#123)`
- Group related changes together
- Be specific but concise
- Link to documentation for complex changes

### Example Entry

```markdown
## [2.1.0] - 2025-01-15

### Added

- New `--format` flag for customizable output (#45)
- Support for YAML configuration files
- Auto-completion for bash and zsh

### Changed

- Default output format changed from JSON to table
- Improved error messages with suggestions

### Fixed

- Fix crash when config file is empty (#42)
- Resolve timeout issues on slow networks (#38)
```
